# JD-focused next steps

This keeps the next phase anchored to the main role, not to a full CRM rebuild.

## Main job to protect

The core role is still:
- analytics
- campaign tracking
- attribution quality
- leadership-ready reporting
- visibility into lead and referral performance

The status project matters because it affects reporting trust, but it should stay in service of the main job.

## Immediate next steps

### 1. Restore access or get a 30-day export
- Ask Avi or IT for the current working CRM access path or a 30-day export.
- Use the field list in [../01_reporting_system/crm_30_day_report_spec.md](../01_reporting_system/crm_30_day_report_spec.md).

### 2. Produce the first 30-day CRM report
- Focus on:
  - lead volume
  - referral volume
  - source and campaign coverage
  - referral-source visibility
  - follow-up gaps
  - status-bucket trust issues

### 3. Identify the first JD-value wins
- missing source cleanup
- missing campaign cleanup
- referral-source cleanup
- overdue follow-up watchlist
- any report where status labels are causing obvious misreads

### 4. Keep status work plain language and narrow
- document current status
- document proposed status
- list unresolved questions
- do not overbuild architecture, dashboards, or automation before the business meaning is confirmed

## What success looks like in the next pass

- You can explain the current lifecycle in plain English.
- You can say which CRM metrics are safe enough to use now.
- You can show a 30-day view of attribution and follow-up health.
- You can separate true analytics work from owner-held status decisions.
- You can bring Avi a shorter, clearer set of unresolved questions.

## Decision rule

If a task improves:
- source tracking
- campaign visibility
- referral visibility
- export trust
- or follow-up discipline

it is probably aligned with the main JD.

If a task expands into:
- deep status architecture
- speculative automation
- final UI design
- or a large technical management system

it is probably too early unless Avi explicitly asks for it.
